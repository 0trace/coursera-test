Anatomy of CSS Rule

[element-selector] {
  color: blue;
}

CSS rule consists of selector and one or more CSS declarations, enclosed in curly braces. CSS declaration contains of property and the value. Collection of CSS rules is a CSS stylesheet.

Element, Class and id Selectors

.classname {
  color: blue;
}

#id-name {
  color: blue;
}

Combining Selectors

[element-selector].classname {
  color: blue;
}

[selector-parent] > [selector-child] {
  color: blue;
}

Child selector combination of selectors -- selects every [selector-child], being direct child of [parent-selector].

[selector-parent] [selector-child] {
  color: blue;
}

Descendant selector combination of selectors -- applies rule to every [selector-child], being child at any level of descendancy to the [parent-selector].

[selector1] + [selector2] {
  color: blue;
}

Adjacent sibling selector

[selector1] ~ [selector2] {
  color: blue;
}

Conflict Resolution

  • Origin Precedence
  • Merge
  • Inheritance
  • Specificity

Origin Precedence and Merge

When to declarations are in conflict -- The last declaration wins. When there is no conflict -- Declarations merge.

Inheritance

All discendents inherit property applied to it's ancestor.

Specificity

The most specific selector wins. Every attribute has a specificity score. The most specific selector is an inline styling, which is not realy a selector. It has a score 1000. The next in line is an id selector with the score 100. A class selector has 10 and the least specific is an ordinary element. In case of combination of selectors we just add scores.

[element-selector] {
  color: blue !important;
}

"!important" defines property, overriding any specificity.
